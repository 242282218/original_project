# Test Release Workflow

来源 Skill: `.trae/skills/test-release/SKILL.md`

## 使用场景

- 版本发布
- 合并
- 上线前

## 执行步骤

1. **检查已执行测试**
   - 单元测试结果
   - 集成测试结果
   - E2E 测试结果

2. **评估关键回归点**
   - 影响的核心功能
   - 变更范围

3. **识别已知风险**
   - 跳过的测试
   - 已知问题

4. **输出放行评估**
   - 当前测试状态
   - 回归覆盖结论
   - 放行建议（Yes / Conditional / No）

## 禁止事项

- 不修改代码
- 不修改测试
