# Frontend Minimal Diff Workflow

来源 Skill: `.trae/skills/fe-minimal-diff/SKILL.md`

## 使用场景

- 修复前端 bug 或样式问题
- 希望保持最小代码改动时

## 执行步骤

1. **定位问题代码**
   - 分析报错信息
   - 找到相关组件

2. **分析最小修改点**
   - 确定必须修改的代码行
   - 评估影响范围

3. **实施修改**
   - 仅修改与问题直接相关的代码
   - 保持代码风格一致

4. **输出报告**
   - 改动点说明
   - 修改理由

## 禁止事项

- 不格式化全文件
- 不重排 JSX
- 不顺手优化无关代码
