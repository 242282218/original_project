# Test Implement Workflow

来源 Skill: `.trae/skills/test-implement/SKILL.md`

## 使用场景

- 需要补充测试以覆盖关键路径
- 降低回归风险时

## 执行步骤

1. **识别关键测试需求**
   - 高风险逻辑
   - 回归风险点

2. **确定最小测试范围**
   - 只新增与风险直接相关的测试
   - 测试数量最小化

3. **实现测试**
   - 覆盖关键逻辑
   - 不重构生产代码

4. **输出报告**
   - 新增测试说明
   - 覆盖点说明

## 禁止事项

- 不重构业务代码
- 不大规模重写测试
