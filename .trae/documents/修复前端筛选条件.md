## 修复前端筛选条件问题

### 问题总结

1. 前端传递 `cloud_types` 为数组，但 pansou 需要逗号分隔字符串
2. 前端使用 `yidong`，但 pansou 期望 `mobile`
3. 参数名需要从 `cloud_types` 转换为 `cloud_types`（实际相同，但需要序列化）

### 修复方案

修改 `quark_strm/web/src/api/search.ts`：

* 添加参数转换逻辑

* 将数组转换为逗号分隔字符串

* 添加网盘类型映射（yidong → mobile）

### 修改内容

```typescript
export const searchResources = (params: {
  keyword: string
  cloud_types?: string[]
  ...
}): Promise<SearchResponse> => {
  // 转换参数格式适配 pansou API
  const convertedParams: any = {
    kw: params.keyword,
    res: 'merged_by_type',
    src: 'all'
  }
  
  // 转换网盘类型
  if (params.cloud_types && params.cloud_types.length > 0) {
    const typeMapping: Record<string, string> = {
      'yidong': 'mobile',
      'aliyun': 'aliyun',
      'baidu': 'baidu',
      'quark': 'quark',
      'tianyi': 'tianyi',
      '115': '115',
      '123': '123'
    }
    convertedParams.cloud_types = params.cloud_types
      .map(t => typeMapping[t] || t)
      .join(',')
  }
  
  return api.get('/search', { params: convertedParams })
}
```
