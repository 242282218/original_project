## 问题分析

**422 错误原因**:
前端直接调用 `/api/search?kw=xxx&res=merged_by_type`，但后端 FastAPI 接口期望的是 `/api/search?keyword=xxx`，参数名不匹配导致校验失败。

**解决方案**:
需要修改前端 `search.ts`，让它调用后端 API 时使用正确的参数名（`keyword` 而不是 `kw`）。
