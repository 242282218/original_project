# Test Design Workflow

来源 Skill: `.trae/skills/test-design/SKILL.md`

## 使用场景

- 新增功能
- 复杂逻辑
- 准备补充测试前

## 执行步骤

1. **分析目标功能**
   - 理解业务逻辑
   - 识别输入输出

2. **设计测试用例**
   - 正向用例
   - 边界条件
   - 异常场景

3. **评估自动化适合度**
   - 是否适合自动化
   - 自动化成本评估

4. **输出测试设计**
   - 测试用例列表
   - 自动化建议（适合/不适合 + 原因）

## 禁止事项

- 不实现测试代码
- 不修改业务代码
