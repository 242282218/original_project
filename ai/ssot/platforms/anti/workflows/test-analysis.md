# Test Analysis Workflow

来源 Skill: `.trae/skills/test-analysis/SKILL.md`

## 使用场景

- 需要整体了解当前测试体系
- 评估覆盖范围
- 识别测试缺口
- 分析环境差异

## 执行步骤

1. **分析测试类型分布**
   - 单元测试数量
   - 集成测试数量
   - E2E 测试数量

2. **评估覆盖范围**
   - 已覆盖的模块
   - 未覆盖的逻辑

3. **检查测试工具与框架**
   - 使用的测试框架
   - Mock 工具

4. **分析环境差异**
   - 本地环境
   - CI 环境
   - 预发布环境

5. **输出分析报告**
   - 测试现状概览
   - 关键测试缺口
   - 高风险区域
   - 环境差异说明

## 禁止事项

- 不修改代码
- 不修改测试
- 不执行命令
